## 1. 项目背景

### 1.1 现状
- 售后系统使用 Zoho Desk 租户系统
- 用户质量反馈以 Tickets 形式登记
- 已积累 2000+ 手工标注的反馈样本
- 数据涉及产品市场表现，敏感且重要

### 1.2 痛点
- 反馈数据量大，人工分类效率低
- 质量状态缺乏实时监控
- 问题趋势无法提前预判
- 异常情况响应滞后

### 1.3 目标
构建自动化质量反馈分析系统，实现从数据采集到智能预警的完整闭环。

## 2. 产品目标

### 2.1 核心目标
| 目标 | 描述 | 衡量标准 |
|------|------|---------|
| 自动化分类 | 用户反馈自动识别意图并分类 | 分类准确率 ≥ 90% |
| 实时看板 | 质量状态可视化展示 | 数据延迟 < 1 小时 |
| 智能预警 | 异常情况自动推送 | 预警覆盖率 100% |
| 趋势预测 | 质量问题提前预判 | 预测准确率 ≥ 70% |

### 2.2 用户画像
| 角色 | 需求 | 使用场景 |
|------|------|---------|
| 质量经理 | 全局质量状态、趋势分析 | 周会汇报、月度复盘 |
| 产品经理 | 功能缺陷分布、用户痛点 | 版本规划、优先级排序 |
| 运维人员 | 实时异常、紧急问题 | 日常监控、应急响应 |

### 5.3 数据模型（简化）

```sql
-- 原始反馈表
CREATE TABLE feedback_raw (
    id SERIAL PRIMARY KEY,
    zoho_ticket_id VARCHAR(50) UNIQUE,
    subject TEXT,
    content TEXT NOT NULL,
    created_at TIMESTAMP,
    product_version VARCHAR(20),
    customer_id VARCHAR(50),
    priority VARCHAR(20),
    status VARCHAR(20),
    raw_data JSONB,
    fetched_at TIMESTAMP DEFAULT NOW()
);

-- 分类结果表
CREATE TABLE feedback_classified (
    id SERIAL PRIMARY KEY,
    feedback_id INTEGER REFERENCES feedback_raw(id),
    primary_category VARCHAR(50),
    secondary_category VARCHAR(50),
    confidence FLOAT,
    intent_summary TEXT,
    keywords TEXT[],
    needs_review BOOLEAN DEFAULT FALSE,
    reviewed_by VARCHAR(50),
    reviewed_at TIMESTAMP,
    model_version VARCHAR(20),
    classified_at TIMESTAMP DEFAULT NOW()
);

-- 统计快照表
CREATE TABLE stats_snapshot (
    id SERIAL PRIMARY KEY,
    snapshot_date DATE,
    category VARCHAR(50),
    count INTEGER,
    period_type VARCHAR(20), -- daily/weekly/monthly
    created_at TIMESTAMP DEFAULT NOW()
);

-- 预警记录表
CREATE TABLE alert_history (
    id SERIAL PRIMARY KEY,
    alert_type VARCHAR(50),
    alert_title TEXT,
    alert_content TEXT,
    severity VARCHAR(20), -- info/warning/critical
    triggered_by TEXT,
    notified_channels TEXT[],
    acknowledged BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```
