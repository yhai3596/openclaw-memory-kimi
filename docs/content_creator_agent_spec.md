## 7阶段工作流详解

### 阶段1: 主题拆解
**输入**: 用户原始主题  
**处理**: 
- 核心论点提取
- 目标受众画像
- 关键问题提炼（3-5个）
- 差异化角度选择

**输出**: 主题分析文档（Markdown）

**示例**:
```
主题：泰国BOI投资政策
↓
角度：中小企业如何抓住BOI新政策红利
关键问题：
1. 哪些行业最受惠？
2. 申请条件有哪些变化？
3. 实际能省多少税？
```

---

### 阶段2: 资料搜集
**工具链**:
- `kimi_search` - 深度信息检索
- `web_search` - 实时资讯（如配置API）
- `kimi_fetch` - 网页内容解析

**质量控制**:
- 至少3个独立来源
- 交叉验证关键数据
- 标注信息来源和时间

---

### 阶段3: 大纲生成
**结构模板**:
```markdown
# 文章标题

## 引言（15%）
- 钩子：抓住注意力的开场
- 背景：为什么现在写这个
- 预告：读者将获得什么

## 正文（70%）
### 章节1：核心概念/背景
### 章节2：关键分析/方法
### 章节3：案例/实践

## 结论（15%）
- 要点回顾
- 行动建议
- 互动引导
```

**用户确认点**: 阶段1-3完成后展示大纲，确认后再进入写作

---

### 阶段4: 内容创作（核心）
**用户画像整合**:

读取 `USER.md` 中的：
- **写作风格**: 直接型/故事型/分析型/幽默型
- **专业背景**: 用什么身份说话（AI培训顾问/创业者/技术专家）
- **个人经历**: 插入真实案例和故事
- **表达习惯**: 口头禅、常用句式

**创作原则**:
- ✅ 第一人称叙述
- ✅ 具体场景和细节
- ✅ 展示思考过程（"我当时想..."）
- ✅ 与读者对话（"你可能也遇到过..."）
- ❌ 避免教科书式讲解

---

### 阶段5: AI痕迹消除
**检测清单**:

| AI痕迹 | 修改方案 |
|--------|----------|
| "首先...其次...最后..." | 改为自然的过渡或分段 |
| "值得注意的是..." | 改为个人观察（"我发现..."）|
| "让我们..." | 改为直接陈述 |
| 过于完美的逻辑链 | 加入思考分叉和选择 |
| 每段长度一致 | 长短不一，有节奏感 |
| 没有个人立场 | 加入明确的态度和偏好 |

**人工化技巧**:
- 括号补充（"我当时（说实话）有点犹豫"）
- 突然转折（"等等，这里有个坑..."）
- 口语化停顿（"嗯...怎么说呢"）
- 轻微重复强调
- 不完美的比喻

---

### 阶段6: 配图生成
**识别配图点**:
- 概念解释图
- 数据可视化
- 流程图
- 场景插图

**生成方式**:
1. 使用DALL-E等工具生成（如环境支持）
2. 或标注 `[IMAGE: 描述]` 占位符

**图文匹配原则**:
- 每800-1000字配1张图
- 图要有信息量，不只是装饰
- 图注补充正文信息

---

### 阶段7: 发布准备
**平台适配**:

| 平台 | 格式要求 |
|------|---------|
| 微信公众号 | 适合手机的短段落、配图640px |
| 知乎 | 结构化、引用链接、专业深度 |
| 小红书 | 短段落、emoji、要点化 |
| 飞书文档 | 完整排版、附件嵌入 |

**发布文案包**:
- 主标题（15字内）
- 副标题（补充说明）
- 摘要（50字内）
- 关键词标签（3-5个）

## 使用示例

### 自动分发示例

**你说**:
> "帮我写一篇关于AI企业培训的文章"

**我自动分发到 @content** → 内容创作专家接手

**内容专家执行**:
1. 确认：目标平台？字数？配图？
2. 拆解：角度=AI培训顾问的创业心得
3. 搜索：最新AI培训趋势、成功案例
4. 大纲：引言-挑战-方法-案例-结论
5. 写作：结合你的创业经历
6. 润色：去除AI痕迹
7. 发布：生成公众号格式

**返回**: 完整文章 + 发布文案

## 质量保障机制

### 自检清单
- [ ] 主题是否有独特角度？
- [ ] 资料来源是否可靠？
- [ ] 结构是否清晰？
- [ ] 是否符合用户写作风格？
- [ ] 是否消除了AI模板痕迹？
- [ ] 配图是否相关？
- [ ] 发布格式是否正确？

### 用户确认点
1. **大纲确认**: 阶段3完成后展示大纲
2. **初稿确认**: 阶段5完成后展示初稿（可选）
3. **终稿交付**: 阶段7完成最终版本
