## 三、热门应用场景与落地案例

**案例来源**：腾讯云开发者社区

**配置规模**：
- 16个AI Agent
- 运营13个自媒体平台
- 27个定时任务

**Agent分工**：
| Agent名称 | 负责平台 | 核心任务 |
|----------|---------|---------|
| 墨媒 | 全局统筹 | 数据监控、日报生成 |
| 墨笔 | 内容创作 | 核心稿件撰写 |

### 3.3 开发者工作流

**典型 workflow**：
```
需求理解 → 技能搜索(find-skills) → 代码开发 → 
PR审查(github) → 文档总结(summarize) → 部署监控
```

**安全警告**：
- 研究发现 ClawHub 约 **15%** 的社区技能包含恶意指令
- 建议：**安装前必须用 skill-vetter 审查**
- 恶意行为模式：数据窃取、凭据收集、后门植入

## 四、技能安全与审查建议

### 4.1 安全风险评估
| 风险等级 | 比例 | 主要问题 |
|---------|------|---------|
| 高风险 | 15% | 恶意代码、数据窃取 |
| 中风险 | 30% | 权限过大、网络请求 |
| 低风险 | 55% | 纯指导型、无代码执行 |

### 4.2 安全审查 checklist
- [ ] 代码是否包含 `eval()` / `exec()` 危险操作
- [ ] 是否请求访问敏感文件（~/.ssh, MEMORY.md等）
- [ ] 是否有未声明的网络请求
- [ ] 是否请求系统级权限
- [ ] 代码是否混淆/压缩

### 4.3 推荐安全实践
1. **强制审查**：所有技能安装前使用 skill-vetter
2. **隔离运行**：在虚拟机或容器中使用
3. **权限最小化**：仅授予必要权限
4. **来源可信**：优先选择官方或高星社区技能
