## 什么是 automation-workflows？

简单来说，这是 ClawHub 上评分高达 **3.729** 的王牌 Skill（满分4.0），专门帮你设计和实施自动化工作流。

它的核心能力可以概括为四点：

| 能力 | 说明 | 适合场景 |
|------|------|---------|
| **机会识别** | 帮你发现哪些重复工作可以自动化 | 每天手动做超过3次的事 |
| **工作流设计** | 把复杂流程拆解成可执行的步骤 | 跨多个工具协作的任务 |
| **工具选型** | 推荐 Zapier、Make、n8n 等自动化平台 | 不同预算和技术栈 |
| **测试维护** | 确保自动化稳定运行，出问题能排查 | 生产环境的关键流程 |

这个 Skill 的触发词设计得很贴心：只要你说了 " automate "、" automation "、" workflow automation "、" save time "、" reduce manual work "，它就会自动介入。

## 实际应用场景：我是怎么用的

作为一个企业AI培训顾问，我有几个高频重复场景：

### 场景1：客户咨询自动分流

以前：收到客户咨询 → 手动回复确认收到 → 询问需求 → 安排会议 → 发送资料

现在：客户咨询 → OpenClaw 自动识别意图 → 分类（培训咨询/服务咨询/合作咨询）→ 自动发送对应资料包 → 我的日历自动弹出可预约时间

**省下的时间**：每天1.5小时

### 场景2：课程素材自动整理

以前：搜集资料 → 手动分类 → 整理成PPT → 检查格式一致性

现在：输入主题 → automation-workflows 自动搜索相关资料 → 按模板整理 → 生成初稿 → 我只需要审核和微调

**省下的时间**：每门课程节省8小时

### 场景3：内容发布自动化

以前：写完文章 → 复制到公众号后台 → 调整格式 → 找配图 → 设置定时 → 手动同步到其他平台

现在：写完文章 → 一键触发工作流 → 自动排版、配图、多平台同步发布

**省下的时间**：每周5小时

## 为什么这个 Skill 值得用？

我用了一段时间后，发现 automation-workflows 有几个特别打动我的点：

### 1. 它不只是工具，是方法论

很多自动化工具只告诉你 "怎么做"，但这个 Skill 会帮你思考 "为什么做" 和 "值不值得做"。它会引导你分析：

- 这个任务自动化的 ROI 是多少？
- 手动和自动的边界在哪里？
- 哪些环节必须人工介入？

这种思维训练，比单纯学会用工具更有价值。

### 2. 覆盖完整的生命周期

从发现自动化机会 → 设计工作流 → 选择工具 → 实施测试 → 长期维护，它是一个完整的闭环。不会出现 "装完就忘、坏了不会修" 的尴尬。

### 3. 适配不同的技术栈

不管你用 Zapier（简单易用）、Make（灵活强大）还是 n8n（开源自主），它都能给出针对性的建议。这点对技术能力参差不齐的创业者很友好。

## 使用建议：从哪开始？

如果你也想尝试 automation-workflows，我的建议是：

**第一步：记录一周的时间日志**

把每天做的事情记下来，标记哪些是重复的、低价值的。通常你会发现，**30%的工作根本不该由你来做**。

**第二步：选一个 "痛点场景" 开刀**

不要贪多，先选一个每天花你最多时间、但又不得不做的任务。比如我选的是 "客户咨询回复"。

**第三步：用 Skill 设计工作流**

在 OpenClaw 对话中触发 automation-workflows，描述你的场景，让它帮你设计完整的自动化方案。

**第四步：小步快跑，持续迭代**

先自动化50%，跑通后再优化到80%，最后争取90%。完美是进步的敌人，先让流程转起来最重要。
